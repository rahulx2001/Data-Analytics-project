Project Overview

InsightFlow is an end-to-end data warehouse project that demonstrates enterprise-grade ETL practices using PostgreSQL. The project implements the Medallion Architecture pattern to progressively refine raw data into analytics-ready datasets.

Key Objectives

  • Integrate data from multiple source systems (CRM & ERP)
  • Apply data cleansing and standardization rules
  • Build a star schema for business intelligence
  • Implement data quality validation checks

Bronze Raw Data Layer

Raw data ingested as-is from source systems without any transformation

Purpose

The Bronze layer serves as the landing zone for all source data. Data is loaded exactly as received from the source systems using PostgreSQL's COPY command for efficient bulk loading.

Bronze layer maintains full data lineage - you can always trace back to the original source data.

Silver Cleansed Data Layer

Data cleansing, standardization, and business rule application

Purpose

The Silver layer represents cleansed, standardized, and trusted data. This layer sits between the raw Bronze layer and the analytical Gold layer, applying business rules and data quality transformations.

Customer Transformations

silver.crm_cust_info

Deduplication

Keep only the latest record per customer using ROW_NUMBER()

ROW_NUMBER() OVER (
    PARTITION BY cst_id 
    ORDER BY cst_create_date DESC
) AS rn
-- Keep WHERE rn = 1

Whitespace Trimming

Remove leading/trailing spaces from names

TRIM(cst_firstname) AS first_name,
TRIM(cst_lastname) AS last_name

Gender Standardization

Convert codes to readable values

CASE
  WHEN UPPER(TRIM(cst_gndr)) = 'F' THEN 'Female'
  WHEN UPPER(TRIM(cst_gndr)) = 'M' THEN 'Male'
  ELSE 'n/a'
END AS gender
'M' 'Male'

Marital Status Standardization

Convert abbreviations to full text

CASE
  WHEN UPPER(TRIM(cst_marital_status)) = 'S' 
       THEN 'Single'
  WHEN UPPER(TRIM(cst_marital_status)) = 'M' 
       THEN 'Married'
  ELSE 'n/a'
END AS marital_status
'S' 'Single'

Product Transformations

silver.crm_prd_info

Category ID Extraction

Extract category ID for ERP joins

REPLACE(
  SUBSTRING(prd_key FROM 1 FOR 5), 
  '-', '_'
) AS cat_id
'AC-HE-123' 'AC_HE'

Product Line Expansion

Convert codes to readable names

CASE
  WHEN UPPER(TRIM(prd_line)) = 'M' THEN 'Mountain'
  WHEN UPPER(TRIM(prd_line)) = 'R' THEN 'Road'
  WHEN UPPER(TRIM(prd_line)) = 'T' THEN 'Touring'
  WHEN UPPER(TRIM(prd_line)) = 'S' THEN 'Other Sales'
  ELSE 'n/a'
END AS product_line

SCD-2 End Date Fix

Calculate end dates using LEAD() for slowly changing dimensions

(LEAD(prd_start_dt) OVER (
    PARTITION BY prd_key 
    ORDER BY prd_start_dt
) - INTERVAL '1 day')::DATE AS prd_end_dt

Null Cost Handling

Replace NULL costs with 0

COALESCE(prd_cost, 0) AS cost

Sales Transformations

silver.crm_sales_details

Date Conversion

Convert integer dates (YYYYMMDD) to DATE type

CASE 
  WHEN sls_order_dt = 0 
    OR LENGTH(sls_order_dt::text) <> 8 
  THEN NULL
  ELSE TO_DATE(sls_order_dt::text, 'YYYYMMDD') 
END AS order_date
20110105 2011-01-05

Sales Recalculation

Enforce: Sales = Quantity × Price

CASE
  WHEN sls_sales IS NULL 
    OR sls_sales <= 0
    OR sls_sales <> sls_quantity * ABS(sls_price)
  THEN sls_quantity * ABS(sls_price)
  ELSE sls_sales
END AS sales_amount

ERP Data Transformations

Customer ID Cleanup

Remove NAS prefix for CRM-ERP joins

CASE 
  WHEN cid LIKE 'NAS%' 
  THEN SUBSTRING(cid FROM 4) 
  ELSE cid 
END AS customer_id
'NAS12345' '12345'

Country Code Expansion

Convert country codes to full names

CASE
  WHEN TRIM(cntry) = 'DE' THEN 'Germany'
  WHEN TRIM(cntry) IN ('US','USA') 
       THEN 'United States'
  WHEN cntry IS NULL OR TRIM(cntry) = '' 
       THEN 'n/a'
  ELSE TRIM(cntry)
END AS country

Birth Date Validation

Replace future dates with NULL

CASE 
  WHEN bdate > CURRENT_DATE 
  THEN NULL 
  ELSE bdate 
END AS birth_date

CREATE VIEW gold.dim_customers AS
SELECT
    ROW_NUMBER() OVER (ORDER BY cst_id) AS customer_key,
    ci.cst_id            AS customer_id,
    ci.cst_key           AS customer_number,
    ci.cst_firstname     AS first_name,
    ci.cst_lastname      AS last_name,
    ci.cst_marital_status AS marital_status,
    CASE 
        WHEN ci.cst_gndr <> 'n/a' THEN ci.cst_gndr  -- CRM is master gender
        ELSE COALESCE(ca.gen, 'n/a')
    END AS new_gen,
    ci.cst_create_date   AS create_date,
    ca.bdate             AS birth_date,
    la.cntry             AS country
FROM silver.crm_cust_info ci
LEFT JOIN silver.erp_cust_az12 ca ON ci.cst_key = ca.cid
LEFT JOIN silver.erp_loc_a101 la ON ci.cst_key = la.cid;

CREATE VIEW gold.dim_products AS
SELECT
    ROW_NUMBER() OVER (ORDER BY pn.prd_start_dt, pn.prd_key) AS product_key, -- Surrogate key
    pn.prd_id            AS product_id,
    pn.prd_key           AS product_number,
    pn.prd_nm            AS product_name,
    pn.cat_id            AS category_id,
    pc.cat               AS category,
    pc.subcat            AS subcategory,
    pc.maintenance       AS maintenance,
    pn.prd_cost          AS cost,
    pn.prd_line          AS product_line,
    pn.prd_start_dt      AS start_date
FROM silver.crm_prd_info pn
LEFT JOIN silver.erp_px_cat_g1v2 pc ON pn.cat_id = pc.id
WHERE prd_end_dt IS NULL; -- filter out historical data

-- Fact Table
CREATE VIEW gold.fact_sales AS
SELECT
    sd.sls_ord_num       AS order_number,
    pr.product_key       AS product_key,
    cu.customer_key      AS customer_key,
    sd.sls_order_dt      AS order_date,
    sd.sls_ship_dt       AS shipping_date,
    sd.sls_due_dt        AS due_date,
    sd.sls_sales         AS sales_amount,
    sd.sls_quantity      AS quantity,
    sd.sls_price         AS price
FROM silver.crm_sales_details sd
LEFT JOIN gold.dim_products pr ON sd.sls_prd_key = pr.product_number
LEFT JOIN gold.dim_customers cu ON sd.sls_cust_id = cu.customer_id;
